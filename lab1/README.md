## Структура
-  `/exp_note` - основной файл с исполняемым jupyter notebook
-  `/all_code` - аналогичный файл .py
-  `/kernels` - реализация Ядер
-  `/metrics` - реализация функций расстояний, метрик(дистанции)
-  `/scores` - реализация метрик(оценки точности)
-  `/knn_parzen` - основной класс классификатора
-  `/model_selection` - хранит реализации LOO, cros_val_score, train_test_split
-  `/main` - основной код запуска(реализовано нахождения лучшего k, рисование графиков, итоговую оценку классификатора), можно запускать его вместо all_code.py(результат аналогичек)


## От себя
Получилось немного странно, так как код находится в 3 местах сразу и копирует друг друга.
- 1 - `/exp_note` чтобы были графики в .ipynb
- 2 - `/all_code` Тоже самое но единным python файлом(сам не знаю для чего)
- 3 - Это все остальные файлы. Все разбито по логическим блокам(например в kernels лежит реализация ядер). Потом все это вызывается из main.py и он становится аналогичен all_code.py файлу.
